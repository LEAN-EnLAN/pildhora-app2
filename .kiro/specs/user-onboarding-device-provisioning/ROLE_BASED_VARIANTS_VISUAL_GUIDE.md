## Best Practices

### âœ… DO

1. **Use the hook for role detection**
   ```typescript
   const { isPatient } = useUserRole();
   ```

2. **Use permission utilities for feature checks**
   ```typescript
   if (canManageDevices(user)) { ... }
   ```

3. **Check device ownership before showing device features**
   ```typescript
   {ownsDevice(user) && <DevicePanel />}
   ```

4. **Use role-based routing**
   ```typescript
   router.push(getHomeRoute(user));
   ```

### âŒ DON'T

1. **Don't check role directly in components**
   ```typescript
   // Bad
   if (user?.role === 'patient') { ... }
   
   // Good
   if (isPatient) { ... }
   ```

2. **Don't duplicate permission logic**
   ```typescript
   // Bad
   if (user?.role === 'patient' && user?.deviceId) { ... }
   
   // Good
   if (canManageDevices(user)) { ... }
   ```

3. **Don't hardcode routes**
   ```typescript
   // Bad
   router.push('/patient/home');
   
   // Good
   router.push(getHomeRoute(user));
   ```

## Troubleshooting

### Issue: Features not showing for patient with device

**Check:**
1. Is `user.deviceId` set in auth state?
2. Is `canManageDevices(user)` returning true?
3. Is the component using the correct permission check?

### Issue: Caregiver seeing patient-only features

**Check:**
1. Is `user.role` correctly set to 'caregiver'?
2. Are permission checks using the utilities?
3. Is there a fallback rendering patient variant?

### Issue: Settings screen not rendering correctly

**Check:**
1. Is `RoleBasedSettings` imported correctly?
2. Is Redux auth state populated?
3. Are there any console errors?
